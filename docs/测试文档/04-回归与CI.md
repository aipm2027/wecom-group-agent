# 测试文档 · 回归与 CI

> 自动化回归与持续集成方案。返回 [测试总览](00-总览.md)

## 1. GitHub Actions CI（已启用）

配置：[.github/workflows/ci.yml](../../.github/workflows/ci.yml)

- **触发**：push 或 pull_request 到 `main` 分支。
- **矩阵**：Python `3.9` / `3.10` / `3.11` / `3.12`（项目在 3.9.6 开发，须兼容 3.9）。
- **步骤**：`actions/checkout@v4` → `actions/setup-python@v5` → 遍历跑 `tests/test_*.py`，任一失败即 job 失败（退出码传播）。
- **零依赖**：核心纯标准库，CI 无需 `pip install`；新增测试文件自动纳入（通配 `tests/test_*.py`）。

```yaml
- name: Run tests
  run: |
    for f in tests/test_*.py; do
      echo "== $f =="
      python "$f" || exit 1
    done
```

## 2. 回归门禁

- **提交前**本地跑全部离线测试：`for f in tests/test_*.py; do python3 "$f"; done`，须全绿（当前 13 套）；再跑 `python3 evals/run_evals.py`（离线金标评测，CI 同步执行）。
- 改动核心 `core/` 或适配器后，至少跑相关离线测试 + `MOCK=1 HANDLER=echo` 管道回归。
- 兼容性：新代码须兼容 Python 3.9（顶部 `from __future__ import annotations`；联合类型只出现在注解里；不用 `match` 语句）。
