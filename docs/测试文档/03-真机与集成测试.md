# 真机与集成测试

本文档描述 `wecom-group-agent` 的手动真机测试步骤、已知验证结果及待补测试项，面向接手项目的工程师。

---

## 1. 手动真机测试步骤

### 1.1 准备 `.env`

```bash
cp .env.example .env
# 编辑 .env，填入 LLM_API_KEY（当前用 StepFun）
```

`.env` 已被 `.gitignore` 忽略，**切勿提交密钥**。

### 1.2 交互式真机测试（Mac 可跑）

```bash
MOCK=1 HANDLER=llm python3 main.py
```

输入格式：

```
chat_id|sender|content
```

示例：

```
群A|张三|@bot 你好
私聊-客户A|李四|那个每日坚果礼盒多少钱
```

> 内容含 `@bot` 视为在群里 @ 了机器人（会被剥掉后再进内容）。

### 1.3 脚本式批量测试

```bash
MOCK=1 HANDLER=llm MOCK_SCRIPT=examples/sample.json python3 main.py
```

脚本文件格式见 `[examples/sample.json](../../examples/sample.json)`，每条是一个 `dict`，可含 `Message` 的任意字段。

### 1.4 断网降级验证

1. 正常启动：`MOCK=1 HANDLER=llm python3 main.py`
2. 发送一条消息，确认有正常回复。
3. 断网（或临时改 `.env` 中 `LLM_BASE_URL` 为一个无效地址）。
4. 再发送一条消息，应看到回复变为：`"不好意思，我这边有点忙，稍后回复你哈~"`。
5. 观察 stderr 有降级日志，但进程不崩溃。
