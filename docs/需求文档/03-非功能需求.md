# 非功能需求

## 1. 合规与封号风险

| 方案 | 合规性 | 能力 | 风险 | 状态 |
|------|--------|------|------|------|
| 微信客服 API（`WecomKfAdapter`） | 官方合规 | 仅 1:1 单聊，**进不了群** | 无封号风险 | **适配器已实现**（代码就绪，真机公网回调联调待验证） |
| ntwork 企微 hook | 违反企微用户协议 | 可进群，实时收发 | **有封号风险**（用户已知情） | 仅 Windows，当前 stub |
| Mock CLI | 不涉及真实平台 | 本地验证 | 无 | 已跑通 |

- **当前策略**：Mac 上只用 Mock 做全链路验证；真实企微群 hook 必须等到 Windows 环境。**微信客服适配器已落地**（`ADAPTER=kf`），是官方合规的 1:1 私域单聊维护通道，只待公网回调地址做真机联调。
- **用户已知情**：使用 ntwork 方案即表示接受封号风险，项目文档与代码注释中已明确标注。

## 2. 健壮性降级

- LLM 调用链路实行"全降级"：任何网络、API、解析异常都返回兜底话术，不抛未处理异常导致进程崩溃。
- 去重表溢出时自动淘汰最旧记录，不中断服务。
- 未配置 `LLM_API_KEY` 时直接返回兜底话术，不打无效请求。
- EchoHandler（复读机）作为 Handler 的占位实现，在 LLM 不可用时可秒级切换验证链路（`HANDLER=echo`）。
- 微信客服回调兜底：签名/解密失败抛受控 `WeComCryptError`、`sync_msg` 校验 `errcode` 不静默丢消息，回调对腾讯始终回 `success` 防重试风暴。
- 知识检索"永不比全量塞更差"：RAG 出错/无 embedding key 时自动回退全量；结构化商品库文件缺失/损坏时返回空串，不崩。
- 会话存储线程安全：内存版与 SQLite 版的 `get`/`all` 均加锁（api_server 多线程下不丢会话）；SQLite 写库异常兜底不上抛，不中断消息主链路。
- 兼容 Python 3.9；GitHub Actions CI 在 3.9–3.12 上跑全部离线测试（10 套），失败退出码传播。

## 3. 可配置

- 所有行为尽可能通过环境变量配置，无需改代码：
  - 适配器：`ADAPTER`（`kf`）、`MOCK`、`MOCK_SCRIPT`
  - Handler：`HANDLER`（`echo`/`llm`）
  - 知识后端：`KNOWLEDGE_PROVIDER`（`static`/`rag`/`structured`/`hybrid`）
  - 会话存储：`STORE`（`sqlite`）、`SQLITE_PATH`（默认 `data/sessions.db`）
  - LLM：`LLM_API_KEY`、`LLM_BASE_URL`、`LLM_MODEL`、`LLM_TIMEOUT`、`LLM_MAX_HISTORY`、`LLM_SYSTEM_PROMPT`、`LLM_PERSONA_FILE`、`LLM_KNOWLEDGE_FILE`
  - RAG embedding：`LLM_EMBED_URL`、`LLM_EMBED_MODEL`、`RAG_TOP_K`、`RAG_SMALL_KB_MAX`、`RAG_TIMEOUT`
  - 微信客服：`WECOM_CORP_ID`、`WECOM_KF_SECRET`、`WECOM_CALLBACK_TOKEN`、`WECOM_ENCODING_AES_KEY`、`WECOM_CALLBACK_PORT`、`WECOM_CALLBACK_PATH`
  - REST API：`ADMIN_TOKEN`、`API_HOST`、`API_PORT`
- `.env` 文件支持（[main.py](../../main.py) 的 `load_env_file`），纯标准库实现，不覆盖已有环境变量。
- `.env` 已加入 `.gitignore`，防止密钥泄露。
- 注：以上环境变量均可在 `.env.example` 找到示例（含 `KNOWLEDGE_PROVIDER`/`STORE`/`SQLITE_PATH`/`ADMIN_TOKEN`/`API_HOST`/`API_PORT`/`LLM_EMBED_*` 等）。

## 4. 隐私与密钥

- API key 仅存于环境变量与 `.env`（不被 Git 追踪）。
- LLM 错误日志只打印状态码、错误类型和响应片段，**绝不打印请求头/请求体**，避免 key 泄露。
- `raw` 字段保留原始 payload 供调试，但注意清理日志中可能包含的敏感信息。
