## 1. Message 规范化模型

`core/message.py` 定义了全链路唯一的消息数据结构，屏蔽 mock / ntwork hook 等不同传输层的字段差异。

```python
@dataclass
class Message:
    chat_id: str                      # 群/会话唯一 id
    chat_type: str                    # "group" | "single"
    msg_id: str                       # 用于去重
    sender_id: str                    # 发送者 id
    sender_name: str                  # 发送者昵称
    content: str                      # 文本内容
    msg_type: str = "text"            # 先只处理 text
    is_at_bot: bool = False           # 群里是否 @ 了机器人
    timestamp: int = 0                # 毫秒
    raw: dict[str, Any] = field(default_factory=dict)  # 原始 payload，留存
```

### 字段语义表

| 字段 | 类型 | 说明 |
|------|------|------|
| `chat_id` | `str` | 群/会话唯一标识，由适配器按传输层原字段映射（如企微 `conversation_id`） |
| `chat_type` | `"group" \| "single"` | 区分群聊与单聊，决定路由触发策略 |
| `msg_id` | `str` | 消息唯一 id，用于 `Router` 去重；mock 模式下由适配器自增生成 |
| `sender_id` | `str` | 发送者唯一标识；`BOT_SENDER_ID = "__bot__"` 表示机器人自身 |
| `sender_name` | `str` | 发送者昵称，LLM handler 会在历史记录中拼接为 `"昵称：内容"` 送入 user role |
| `content` | `str` | 文本内容；若为空则在 LLM handler 组装时会被跳过 |
| `msg_type` | `str` | 当前仅支持 `"text"`，非 text 消息触发策略直接拒绝 |
| `is_at_bot` | `bool` | 群聊中是否 @ 了机器人；`MockCliAdapter` 将内容中的 `@bot` 解析为此标记 |
| `timestamp` | `int` | 毫秒时间戳，当前主要留存，未参与限流逻辑 |
| `raw` | `dict` | 原始传输层 payload，供调试或未来扩展字段时使用 |

```python
@property
def is_group(self) -> bool:
    return self.chat_type == "group"
```

## 2. Router.on_message 完整流程

`core/router.py` 的 `Router.on_message` 是整条链路入口。流程固定为：

**去重 → 入会话 → 接管静默检查 → 触发判断 → 限流 → handler 生成回复 → 回发 → 记忆 → on_escalate 回调**

```python
def on_message(self, msg: Message) -> None:
    # 1) 去重（重复推送直接丢）
    if msg.msg_id and self._is_duplicate(msg.msg_id):
        return

    # 记入会话上下文（无论是否回复，都保留历史）
    session = self.sessions.get(msg.chat_id)
    session.add(msg)

    # 人工已接管：agent 静默，只记录消息、不自动回复
    if session.human_controlled:
        return

    # 2) 触发判断
    if not should_respond(msg):
        return

    # 3) 限流
    now = time.monotonic()
    if self._rate_limited(msg.chat_id, now):
        return

    # 4) 生成回复（handler 可能在 session 上标记 needs_human）
    reply = self.handler.reply(msg, session)
    if not reply:
        return

    # 5) 回发（先发成功，再更新限流时间戳 & 记入会话上下文）
    self.adapter.send(msg.chat_id, reply)
    self._last_reply_at[msg.chat_id] = now
    session.add(Message(
        chat_id=msg.chat_id,
        chat_type=msg.chat_type,
        msg_id=f"bot-reply-to-{msg.msg_id}",
        sender_id=BOT_SENDER_ID,
        sender_name="Bot",
        content=reply,
        msg_type="text",
    ))

    # 6) 若本轮 agent 判定需人工，触发升级回调（通知工作台等）
    if session.needs_human and self.on_escalate is not None:
        try:
            self.on_escalate(session)
        except Exception:
            pass
```

### 逐步解释

| 步骤 | 函数/逻辑 | 目的与关键参数 |
|------|-----------|----------------|
| **1. 去重** | `_is_duplicate(msg_id)` | 使用 `OrderedDict` 保存已见 `msg_id`，上限 `DEDUP_MAX = 5000`。命中即丢；新 id 写入，超限时 `popitem(last=False)` 淘汰最旧。 |
| **2. 入会话** | `session.add(msg)` | 无论是否触发回复，**所有消息都记入会话历史**，保证多轮上下文完整。 |
| **2.5 接管静默** | `session.human_controlled` | **已落地。** 人工接管时 Router 只记消息、不调 `handler.reply`，agent 完全静默。 |
| **3. 触发策略** | `should_respond(msg)` | 独立函数，见第 3 节。非 text / 群聊未 @bot 均在此过滤。 |
| **4. 限流** | `_rate_limited(chat_id, now)` | 每个会话独立限流，`MIN_INTERVAL_SEC = 2.0` 秒。`now - last < 2.0` 时直接静默丢弃。 |
| **5. 生成回复** | `handler.reply(msg, session)` | 调用 `EchoHandler` 或 `LLMHandler`。若返回空或 `None`，流程终止。 |
| **6. 回发** | `adapter.send(chat_id, text)` | 通过当前适配器发出；**先发成功，再更新限流时间戳**，避免发送失败却误记时间。 |
| **7. 记忆** | `session.add(bot_msg)` | 将机器人回复以 `sender_id = BOT_SENDER_ID` 写回 `session`，供后续多轮对话上下文使用。 |
| **8. 升级回调** | `on_escalate(session)` | **已落地。** 若 `session.needs_human=True`，调用已注册的 `on_escalate` 回调。回调内部异常被捕获，不中断主链路。 |

### 关键常量

```python
DEDUP_MAX = 5000          # 去重表最多记住多少个 msg_id
MIN_INTERVAL_SEC = 2.0    # 每个会话两次回复之间的最小间隔
```

## 3. 触发策略 should_respond

```python
def should_respond(msg: Message) -> bool:
    if msg.msg_type != "text":
        return False
    if msg.is_group:
        return msg.is_at_bot
    return True
```

### 当前规则

| 场景 | 行为 | 目的 |
|------|------|------|
| `msg_type != "text"` | 不回 | 当前仅处理文本，图片/语音等直接静默 |
| 群聊 (`is_group=True`) | 仅当 `is_at_bot=True` 时回复 | 防刷屏，只在被 @ 时响应 |
| 单聊 (`is_group=False`) | 全回 | 私域 1:1 场景，每条消息都视为客服请求 |

### 如何修改

`should_respond` 是独立顶层函数，与 `Router` 解耦，修改无需动 `on_message` 主流程。

**常见扩展方向：**

- **关键词触发**：在 `msg.is_group` 分支中增加 `or "客服" in msg.content` 等条件。
- **全员回复**：将 `if msg.is_group: return msg.is_at_bot` 改为 `return True`，群聊也全回（适合小群/测试）。
- **白名单/黑名单**：读取 `msg.sender_id` 或 `msg.chat_id` 做过滤。
- **AI 判断**：先调轻量模型判断意图，再决定是否走完整 `handler.reply`。

> 修改后仅影响触发，不影响去重、限流、会话记录等机制。

## 4. Session / SessionStore

`core/session.py` 维护按 `chat_id` 隔离的会话上下文。

```python
@dataclass
class Session:
    chat_id: str
    history: Deque[Message] = field(default_factory=lambda: deque(maxlen=MAX_CONTEXT))
    state: dict = field(default_factory=dict)  # 预留：阶段标记 / 变量等
    human_controlled: bool = False   # True 时人工已接管，agent 静默不自动回
    needs_human: bool = False        # agent 判定需转人工，待人工认领
    escalation_reason: str = ""      # 转人工原因（agent 或规则填）
    # 写穿透钩子：持久化存储（SQLite）挂上后自动落库；内存版留 None
    on_message: Optional[Callable[["Session", Message], None]] = None
    on_flags: Optional[Callable[["Session"], None]] = None

    def add(self, msg: Message) -> None:
        self.history.append(msg)
        if self.on_message:
            self.on_message(self, msg)

    def mark_needs_human(self, reason: str = "") -> None:
        self.needs_human = True
        if reason and not self.escalation_reason:
            self.escalation_reason = reason
        if self.on_flags:
            self.on_flags(self)

    def take_over(self) -> None:
        self.human_controlled = True
        self.needs_human = False
        if self.on_flags:
            self.on_flags(self)

    def release(self) -> None:
        self.human_controlled = False
        self.needs_human = False
        self.escalation_reason = ""
        if self.on_flags:
            self.on_flags(self)

class SessionStore:
    def __init__(self) -> None:
        self._sessions: dict[str, Session] = {}
        self._lock = threading.Lock()   # ThreadingHTTPServer 多线程下保护 get/all

    def get(self, chat_id: str) -> Session:
        with self._lock:
            s = self._sessions.get(chat_id)
            if s is None:
                s = Session(chat_id=chat_id)
                self._sessions[chat_id] = s
            return s

    def all(self) -> list[Session]:
        with self._lock:
            return list(self._sessions.values())
```

### 设计要点

| 项 | 说明 |
|----|------|
| `MAX_CONTEXT = 20` | 每个会话仅保留最近 20 条消息，内存安全。 |
| `history` | `deque(maxlen=20)`，自动淘汰旧消息，无需手动清理。 |
| `state` | 预留字典，未来可存放会话阶段（如"已询价→待下单"）、用户标签等。 |
| `human_controlled` | **已落地。** `True` 时 Router 只记消息、不调 handler，agent 静默。 |
| `needs_human` | **已落地。** 由 LLM 回复中的 `[[转人工]]` 标记触发，用于人工工作台收件箱。 |
| `escalation_reason` | **已落地。** 转人工原因摘要，随 `mark_needs_human(reason)` 写入。 |
| `SessionStore.get()` | 懒创建：首次访问 `chat_id` 时自动实例化 `Session`。内部已加 `threading.Lock`，多线程（如 `ThreadingHTTPServer`）下并发安全。 |
| `SessionStore.all()` | 返回全部会话列表，同样受锁保护。 |

### SQLite 持久化（已实现）

`core/session_sqlite.py` 已实现 `SqliteSessionStore`，与内存 `SessionStore` 保持同接口（`get(chat_id) -> Session`、`all() -> list[Session]`），可零改动替换。

| 设计要点 | 方案 |
|----------|------|
| 接口兼容 | 与内存 `SessionStore` 同接口：`get(chat_id) -> Session`、`all() -> list[Session]`。Router 与 Handler 零改动。 |
| 写穿透 | `Session` 注册 `on_message`/`on_flags` 钩子，`SqliteSessionStore` 在钩子中实时写入 `messages`/`sessions` 表，无需显式 `save()` 调用。 |
| `MAX_CONTEXT` 语义保留 | 查询时按 `ORDER BY id DESC LIMIT ?`（`?`=`MAX_CONTEXT`；自增主键 `id` 近似插入顺序）取最近消息，再按时间正序送入 LLMHandler，与内存 `deque(maxlen=20)` 语义一致。 |
| 去重 | `messages` 表设 `UNIQUE(chat_id, msg_id)`，重复消息自动丢弃。 |
| 重启恢复 | 进程重启后，`SqliteSessionStore.get(chat_id)` 从 SQLite 读取最近 N 条历史重构 `Session`；同时恢复 `human_controlled`/`needs_human`/`escalation_reason` 状态，确保接管/转人工不丢失。 |
| 并发安全 | 使用 `threading.RLock`，保护 `_live` 缓存与数据库操作，可重入安全。 |
| 异常兜底 | `_persist_message` / `_persist_flags` 捕获 `sqlite3.Error` 并打印 `stderr`，不上抛，避免 DB 故障中断 Router 的消息处理主链路。 |
| SQL 参数化 | 所有查询使用参数化占位符，无 SQL 注入风险。 |

源码：`core/session_sqlite.py`（[查看](../../core/session_sqlite.py)）。

## 5. 入口 main.py

`main.py` 负责读取环境变量、装配 Adapter / Handler / Knowledge / Session / Router，然后启动。

def main() -> None:
    load_env_file()

### 5.1 load_env_file

```python
def load_env_file(path: str = ".env") -> None:
    """极简 .env 加载：把 KEY=VALUE 读进 os.environ（不覆盖已存在的）。纯 stdlib。"""
    here = os.path.join(os.path.dirname(os.path.abspath(__file__)), path)
    if not os.path.exists(here):
        return
    with open(here, encoding="utf-8") as f:
        for line in f:
            line = line.strip()
            if not line or line.startswith("#") or "=" not in line:
                continue
            key, _, val = line.partition("=")
            os.environ.setdefault(key.strip(), val.strip().strip('"').strip("'"))
```

- 使用 `os.environ.setdefault`，**已存在的环境变量不会被 `.env` 覆盖**，方便临时调试。
- 纯标准库，不依赖 `python-dotenv`。

### 5.2 build_adapter

### 5.4 build_knowledge（知识模块装配）

```python
def build_knowledge():
    kind = os.environ.get("KNOWLEDGE_PROVIDER", "static").lower()
    if kind == "rag":
        from core.knowledge import RagKnowledgeProvider
        return RagKnowledgeProvider()
    if kind == "structured":
        from core.knowledge import StructuredKnowledgeProvider
        return StructuredKnowledgeProvider()
    if kind == "hybrid":
        from core.knowledge import HybridKnowledgeProvider, StaticKnowledgeProvider, StructuredKnowledgeProvider
        return HybridKnowledgeProvider(StructuredKnowledgeProvider(), StaticKnowledgeProvider())
    return None  # static / 未知 → LLMHandler 默认 StaticKnowledgeProvider
```

- `KNOWLEDGE_PROVIDER=static`（默认）：全量返回 `prompts/knowledge.md`，等价 `StaticKnowledgeProvider`。
- `KNOWLEDGE_PROVIDER=rag`：启用语义检索（RagKnowledgeProvider），支持 embedding 向量召回 + 关键词/子串/常驻块混合检索。若 embedding 服务未配置，自动回退全量知识。
- `KNOWLEDGE_PROVIDER=structured`：启用结构化商品库（StructuredKnowledgeProvider），按 name/product_id/category/keywords 做精确/子串匹配，返回价格/规格/库存/促销/状态。
- `KNOWLEDGE_PROVIDER=hybrid`：组合结构化查商品 + 全量知识兜底（HybridKnowledgeProvider），各自 try/except 兜底。

### 5.5 build_sessions（会话持久化装配）

```python
def build_sessions():
    if os.environ.get("STORE") == "sqlite":
        from core.session_sqlite import SqliteSessionStore
        return SqliteSessionStore(os.environ.get("SQLITE_PATH", "data/sessions.db"))

```

- `STORE=sqlite`：使用 `SqliteSessionStore` 持久化，进程重启可恢复最近 `MAX_CONTEXT` 条历史及接管/转人工状态。`main.py` 与 `api_server.py` 指向同一 `SQLITE_PATH` 即可共享。
- 未配置：内存版 `SessionStore`（进程退出丢失）。

### 5.6 Router 装配与 on_escalate

```python
router = Router(adapter, build_handler(), build_sessions(), on_escalate=_default_on_escalate)
```

- `_default_on_escalate`：默认将转人工信号打印到 `stderr`，生产可替换为 webhook/写工单队列/通知人工工作台。
- `on_escalate` 由 `Router.on_message` 在检测到 `session.needs_human=True` 时触发，发生在回复回发之后，不影响主链路。回调内部异常被捕获，不会中断 Router。
