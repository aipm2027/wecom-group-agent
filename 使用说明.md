# 使用说明(运营者手册)

> 写给**不写代码的运营者**:这套 AI 客服怎么用、怎么换成你自己的店铺、出问题怎么排查。
> 技术细节一概不在本文——工程师请看 [README](README.md) 与 [docs/](docs/技术文档/00-总览.md)。

---

## 1. 它是什么

一个 7x24 自动回复的**私域电商 AI 导购客服**:像懂行的老导购一样答疑、查价、适度种草,拿不准的自动**转人工**并说明原因;你随时可以在网页工作台**接管**任何对话。

**两个接入通道现状**:

| 通道 | 服务对象 | 状态 |
|------|---------|------|
| 🤖 智能机器人「咕咕嘎嘎」 | 企业微信**内部成员**(单聊 + 群里@它) | ✅ **已上线可用** |
| 💬 微信客服 | **外部微信客户**(1:1) | ⏸ 代码就绪,**等企业认证**后按 [联调手册](docs/技术文档/07-真机联调SOP.md) 开通(约 40 分钟) |

## 2. 五分钟上手(和机器人聊天)

1. 打开企业微信 → 搜「**咕咕嘎嘎**」(需在机器人的「可见范围」内,后台可加人);
2. **单聊**直接发消息;**群聊**把它拉进群后 `@咕咕嘎嘎 你的问题`;
3. 试这三句,分别验证三种能力:
   - `每日坚果礼盒多少钱?` → 结构化商品库**精确查价**
   - `坚果受潮了怎么办?` → 店铺知识 FAQ
   - `发霉了我要退款!` → **转人工**(工作台收件箱会出现这条会话,带「售后退款/理赔」标签)

> 没反应?见 [第 7 节排查](#7-常见问题排查)。

## 3. 换成你自己的店铺(核心三件事)

机器人现在卖的是样例店铺「暖食记坚果」。换成你的店,只改三个文件(都是普通文本,用任何编辑器):

| 文件 | 改什么 | 生效方式 |
|------|--------|---------|
| [prompts/knowledge.md](prompts/knowledge.md) | 店铺信息/活动/物流/售后政策/FAQ,**照着原文格式替换内容** | 改完**立即生效**,无需重启 |
| [prompts/products.json](prompts/products.json) | 商品库:名称/价格/规格/库存/关键词。有 ERP 的话用 `python3 examples/products_from_csv.py 你的导出.csv` 一键转换 | 需重启机器人 |
| [prompts/persona.md](prompts/persona.md) | 说话风格/禁忌/转人工规则(**改动要谨慎**,合规红线都在这里) | 需重启机器人 |

**改完怎么验证**(强烈建议,顺序执行):
1. 工作台右上「🧪 试聊调试」发几个真实问题,看回复对不对(不打扰真实客户);
2. 终端跑 `make eval`(免费,秒级)—— 检索层回归;
3. 改过 persona 的话跑 `make gate`(调真模型,约 1 分钟)—— **合规红线门禁,全绿才上线**。

## 4. 运营工作台(网页,接管对话用)

```bash
make api        # 窗口 1:接口服务
make console    # 窗口 2:工作台
```
浏览器打开 `http://127.0.0.1:8090`,口令是 `.env` 里的 `CONSOLE_PASSWORD`。

**界面与日常动作**:
- 左侧「**待人工**」标签(带红色角标)= AI 兜不住的会话,点开可见**转人工原因**(客户点名找人工 / 售后退款理赔 / 情绪激烈投诉 / AI 判定复杂场景);
- 点「**人工接管**」→ AI 立刻闭嘴,你在下方输入框回复客户(Ctrl/⌘+Enter 发送);
- 处理完点「**交回 AI**」恢复自动回复;
- 顶部数字条:会话量/消息量/待人工/已接管,4 秒自动刷新;手机浏览器也能用。

> ⚠️ 人工回复能真实发到企微,前提是该客户 **24 小时内**给机器人发过消息(平台规则)。

## 5. 日常启停与运维

```bash
# 启动机器人(推荐生产形态)
ADAPTER=aibot HANDLER=llm KNOWLEDGE_PROVIDER=hybrid STORE=sqlite python3 main.py

# 停止:Ctrl+C 或 kill <进程号>(会优雅退出,聊天记录在 data/ 不会丢)
make health     # 一键体检:api/工作台/机器人都活着吗
make test       # 全部离线测试(改完代码/配置后跑,15 套应全绿)
make eval       # 知识检索回归(免费)
make gate       # 合规门禁(改 persona/换模型前必跑)
```

Docker 一把梭(服务器上推荐):`docker compose up -d`(agent+接口+工作台三件套),日志 `docker compose logs -f agent`。

**⚠️ 一个大坑**:同一个机器人**同时只能有一个进程在线**。如果你在服务器跑着,又在本机再启一个,两边会每 30 秒互相踢下线、无限循环(日志反复出现"被新连接踢下线")。本地调试前,先停掉服务器上的。

## 6. 配置速查(.env 人话版)

| 配置 | 作用 | 哪里拿 |
|------|------|--------|
| `LLM_API_KEY` | AI 大脑的钥匙(阶跃星辰) | platform.stepfun.com 控制台 |
| `WECOM_AIBOT_ID` / `WECOM_AIBOT_SECRET` | 智能机器人身份凭据 | 企微后台→智能机器人→API 配置,**必须点「复制」按钮,手抄必错** |
| `CONSOLE_PASSWORD` | 工作台登录口令 | 自己设一个 |
| `ADMIN_TOKEN` | 接口服务的钥匙(工作台内部用) | 自己设一个,与上面口令**不要相同** |
| `KNOWLEDGE_PROVIDER=hybrid` | 知识模式:精确查价+FAQ 兜底(推荐,不用动) | — |
| `STORE=sqlite` | 聊天记录落盘,重启不丢(推荐,不用动) | — |

> `.env` 是密钥文件:**永远不会**被提交到 GitHub;截图发人前先遮住;疑似泄露就去企微后台「随机获取」换新 Secret 并同步进 `.env`。

## 7. 常见问题排查

| 现象 | 原因与办法 |
|------|-----------|
| 启动秒退,屏幕有「启动自检」字样 | **这是保护你**:它列出了每一条配置问题和修法,照着改完再启动即可(比如漏填 LLM_API_KEY 时绝不会"假装能用") |
| 机器人不回消息 | 按顺序查:① 进程活着吗(`make health`);② 日志最后一行是"订阅成功"还是在反复重连(`双开互踢`见第 5 节);③ 群聊必须 `@咕咕嘎嘎`,单聊才是全回;④ 你在机器人「可见范围」里吗 |
| 回复驴唇不对马嘴 | 知识库还是样例店铺 → 按第 3 节替换;替换后用试聊面板验证 |
| 客户被莫名转人工 / 该转不转 | 工作台看转人工**标签**判断原因;规则词表在 `core/llm_handler.py` 顶部,语义规则在 `persona.md` 转人工章节;改完跑 `make gate` |
| 工作台登录说口令错误 | 用 `.env` 里 `CONSOLE_PASSWORD` 的值;都没配则任何人免密可进(仅限本机调试时可接受) |
| 人工回复显示"未真实下发" | 该客户超过 24h 没发过消息(平台窗口限制),等客户再来一条即可回 |

## 8. 想开通对外的微信客服?

前提只有一个:**企业微信完成企业认证**(后台首页有「前往认证」入口)。认证过了随时喊工程师(或 AI)照 [07-真机联调SOP](docs/技术文档/07-真机联调SOP.md) 走一遍,全程约 40 分钟,之后外部微信客户扫码即可咨询,和智能机器人共用同一套大脑、知识库与工作台。
